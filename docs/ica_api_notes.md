# ICA API Spike Notes

> Generated by Sprint 0 — `scripts/ica_spike.py`

## Endpoint

```
GET https://tg.i-c-a.su/json/{username}?limit={N}&page={P}
```

## Pagination

- **Confirmed:** `?page=N` works (1-based).
- Page 1 returns newest messages; page 2 returns older messages.
- Order within page: **newest → oldest** (descending `id` / `date`).
- Empty `messages` array signals end of available history.

## Rate Limit

- Observed safe interval: **≥ 4 seconds** between requests.
- Plan uses `ICA_RATE_LIMIT_PER_MIN=15` and `FETCH_PAGINATION_DELAY_SECONDS=4`.
- HTTP 429 observed as `FLOOD_WAIT_N` — spike script waits N+1 seconds and retries.

## Timestamps

- `date`: Unix timestamp (seconds) — used as `published_at` (UTC).
- `edit_date`: Present on edited posts — maps to `messages.edit_date`.
- Timezone: UTC via `datetime.fromtimestamp(..., tz=timezone.utc)`.

## Message Fields

| ICA field | DB mapping |
|-----------|------------|
| `id` | `message_id` |
| `message` | `text` (HTML stripped) |
| `date` | `published_at` |
| `edit_date` | `edit_date` |
| `reply_to.reply_to_msg_id` | `reply_to_message_id` |
| `media` | `media_meta` |

## Media Structure

- `messageMediaPhoto`: `photo.sizes[]` with `w`, `h`.
- `messageMediaDocument`: `document.attributes[]`:
  - `documentAttributeVideo`: `duration`, `w`, `h`
  - `documentAttributeImageSize`: `w`, `h`

## Limits

- `limit` parameter accepted; tested with 3, 50.
- Max practical per page: **50** (config default `FETCH_PAGE_SIZE=50`).
- Use `FETCH_MAX_PAGES=20` for backfill safety (up to 1000 msgs/cycle).

## Fetch Algorithm (Sprint A)

```
page = 1
loop:
  GET /json/{channel}?limit=50&page={page}
  for msg in messages (newest→oldest):
    if published_at < overlap_start: STOP
    else: queue insert
  if empty or reached overlap_start or page >= MAX_PAGES: break
  page += 1; sleep(4s)
```

`overlap_start = min(now - 60min, last_successful_fetch_at)`

## Risks

- ICA is SPOF — `FetcherBackend` abstraction allows Telethon swap.
- MTProto JSON format may change — parser uses defensive access.
